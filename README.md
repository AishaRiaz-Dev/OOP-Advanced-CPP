## 🧠 Core OOP Pillars Implemented

This repository moves beyond basic scripting to demonstrate practical application of the four core pillars of Object-Oriented Programming:

### 1. Encapsulation & Data Hiding
* Restricting direct access to object state by utilizing strict access specifiers (`private`, `protected`, `public`).
* Implementing getter and setter methods to enforce robust data validation boundaries.

### 2. Abstraction
* Splitting complex implementation details away from the user interface using classes and header structures.
* Designing modular system workflows where objects interact through clean, high-level interfaces.

### 3. Inheritance (Code Reusability)
* Utilizing single, multiple, and hierarchical inheritance structures to build efficient, scalable class hierarchies.
* Eliminating redundant code logic by deriving specialized classes from robust base classes.

### 4. Polymorphism (Dynamic & Static Behavior)
* **Compile-time Polymorphism:** Implementing function overloading and operator overloading to customize operational logic.
* **Runtime Polymorphism:** Leveraging `virtual` functions and overriding mechanisms to resolve object behaviors dynamically at execution time.
